# Hooks（カスタムフック）

Claude Code のセッション中に特定のタイミングで実行されるシェルスクリプトです。

## 📋 含まれるフック

### 🛡️ safety-check.sh

**トリガー**: `PreToolUse` - Bash ツール実行前

**目的**: 危険なコマンドの実行を防ぐ

**チェック項目**:
- `rm -rf` - 強制的な再帰削除
- `git push --force` - 強制プッシュ
- `/data2/` への書き込み（データディレクトリの保護）
- その他の破壊的操作

**動作**:
- 危険なコマンドを検出した場合、エラーメッセージを返して実行を中止
- 安全なコマンドの場合は通過

---

### 📁 path-guard.sh

**トリガー**: `PreToolUse` - Write/Edit ツール実行前

**目的**: ファイル書き込み先のパスが適切かチェック

**チェック項目**:
- `/data2/` への書き込みを制限（データストレージは書き込み禁止）
- `/home/yuki0024/packages/` への変更を制限（共有ツールの保護）
- システムディレクトリへの書き込み防止

**動作**:
- 禁止されたパスへの書き込みをブロック
- 許可されたパス（`/home/yuki0024/` など）への書き込みは許可

---

### 📐 conventions.sh

**トリガー**: `PreToolUse` - Write/Edit ツール実行前

**目的**: コーディング規約やファイル命名規則のチェック

**チェック項目**:
- ファイル命名規則の確認
- コーディングスタイルのチェック
- プロジェクト規約への準拠

**動作**:
- 規約違反を検出した場合、警告または中止
- 規約に従っている場合は通過

---

### 🎯 session-context.sh

**トリガー**: `SessionStart` - セッション開始時

**目的**: プロジェクトのコンテキスト情報を提供

**機能**:
- プロジェクトのパス構成を説明
- 重要な規約やルールを表示
- セッション固有の設定を適用

**動作**:
- セッション開始時に情報メッセージを表示
- プロジェクト固有の環境変数や設定を読み込み

---

## 🔧 フックの使用方法

### インストール

```bash
# このリポジトリから Claude 設定ディレクトリにコピー
cp -r hooks/* ~/.claude/hooks/

# 実行権限を付与
chmod +x ~/.claude/hooks/*.sh
```

### 設定ファイルでの登録

`~/.claude/settings.json` の `hooks` セクションで登録:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "$HOME/.claude/hooks/safety-check.sh"
          }
        ]
      }
    ]
  }
}
```

### テスト

各フックを個別にテストできます:

```bash
# safety-check.sh のテスト
echo 'rm -rf /' | ~/.claude/hooks/safety-check.sh
# → エラーメッセージが表示されるはず

# path-guard.sh のテスト
echo '/data2/test.txt' | ~/.claude/hooks/path-guard.sh
# → 警告が表示されるはず
```

## 🔍 フックのカスタマイズ

各フックは通常のシェルスクリプトなので、プロジェクトのニーズに合わせて編集できます。

### フックの入力

- **PreToolUse フック**: ツールの引数が JSON 形式で標準入力から渡される
- **SessionStart フック**: セッション情報が渡される

### フックの出力

- **成功**: 終了コード 0 を返す
- **失敗**: 終了コード 1 を返し、エラーメッセージを標準エラー出力に出力
